Sou desenvolvedor criando um sistema chamado "Cheiraí" para controlar validade
de produtos após abertura.

TECNOLOGIA:
- Backend: Node.js + Express
- Frontend: React + Vite
- Banco: MySQL 8.0+
- Autenticação: JWT + bcryptjs

REQUISITOS:

1. TABELAS:
   - usuario: id, nome, email (UNIQUE), senha, criado_em, atualizado_em
   - categoria: id, nome, icone, criado_em
   - local_armazenamento: id, nome, icone, criado_em
   - produto: id, id_usuario, id_categoria, id_local, nome, data_abertura, 
             prazo_validade, data_vencimento (calculada), status (ENUM),
             codigo_barras, observacao, criado_em
   - historico: id, id_produto, id_usuario, tipo (consumido/descartado),
               estava_vencido, data_acao
   - sugestao_prazo: id, nome_alimento, prazo_dias, id_categoria

2. STATUS AUTOMÁTICO DO PRODUTO:
   - 'ok': mais de 3 dias
   - 'atencao': 1-3 dias
   - 'vence_hoje': hoje
   - 'vencido': já venceu

3. TRIGGERS:
   - Calcular data_vencimento automaticamente
   - Atualizar status baseado em data_vencimento
   - Validar que produtos vencidos só podem ser descartados

4. EVENT:
   - Atualizar status de todos os produtos diariamente à meia-noite

5. DADOS INICIAIS:
   Categorias: Laticínios, Carnes, Frutas, Legumes, Condimentos, 
              Bebidas, Congelados, Padaria, Outros
   Locais: Geladeira, Freezer, Armário, Bancada
   Sugestões: mínimo 20 alimentos com prazos

Gere script SQL COMPLETO, pronto para MySQL Workbench, 100% comentado.
